GitHub Pages Troubleshooting Guide

Solutions for when your GitHub Pages site shows "Page not found"

Repository Not Named Correctly

For user/organization sites, your repository must be named exactly: username.github.io

Rename your repository to match your username exactly:

yourusername.github.io

GitHub Pages Not Enabled

You might not have enabled GitHub Pages for your repository.

Go to your repository Settings ‚Üí Pages ‚Üí Select source branch (usually main or master) ‚Üí Save.

Wrong Branch or Folder

Your files might not be in the correct branch or folder.

Make sure your site files are in the root directory of the selected branch, or in the /docs folder if using that option.

No Index.html File

Your repository might be missing an index.html file.

Add an index.html file to the root of your repository. This is the default file that GitHub Pages will serve.

Custom Domain Configuration Issues

If using a custom domain, there might be DNS configuration problems.

Check your DNS settings and ensure you've added the correct A/AAAA records or CNAME file in your repository.

Jekyll Build Errors

If using Jekyll, there might be errors during the build process.

Check for build errors in your repository Settings ‚Üí Pages ‚Üí Build notifications.

Cache Issues

Your browser or GitHub's CDN might be serving cached content.

Try a hard refresh (Ctrl+F5) or check your site in an incognito window. GitHub Pages may take up to 10 minutes to update.

Repository Visibility

If your repository is private, you need GitHub Pro for GitHub Pages.

Make your repository public or upgrade to GitHub Pro if you want to keep it private.

üí°Quick Setup Checklist

  1. Create a repository named yourusername.github.io (replace with your username)
  2. Add an index.html file to the root of the repository
  3. Enable GitHub Pages in your repository settings
  4. Select the main branch as source
  5. Wait a few minutes for GitHub to build and deploy your site
  6. Visit https://yourusername.github.io
